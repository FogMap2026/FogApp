# 🧭 여행 성향 테스트 설계 (Personality Test Design)

> 소셜 트랙(SOC) Phase 1 선행 작업. 지도/안개 의존성 없이 성향 축·설문·점수 모델을 확정해
> 2-6(테스트 화면 + 결과 저장), 3-8(매칭 알고리즘)이 바로 이 모델 위에서 진행되도록 한다.
> 저장 위치는 [ERD.md](ERD.md)에 정의된 `users.personality_type` / `users.personality_scores(jsonb)`.

---

## 1. 배경 & 목표

[README](../README.md)의 "여행 성향 매칭" 기능은 즉흥성/계획성, 휴양/관광, 외향/내향 등
다각도 성향을 분석해 동행을 추천한다. 이 문서는 그 분석에 필요한:

1. **성향 축**을 확정하고
2. 각 축을 측정하는 **설문 문항**을 설계하고
3. 응답 → **점수·유형으로 변환하는 계산 모델**을 정의한다.

이 세 가지가 정해지면 2-6(UI+저장)과 3-8(매칭 알고리즘)은 이 문서의 모델을 그대로 구현하면 된다.

---

## 2. 성향 축 (Personality Axes)

각 축은 **0~100 점수**로 표현되는 양극 스펙트럼이다. 50을 기준으로 어느 쪽에 가까운지로 유형 코드를 정한다.

| 축 코드 | 이름 | 0에 가까움 | 100에 가까움 |
|---------|------|-----------|--------------|
| `S` | 즉흥성 ↔ 계획성 (Spontaneity) | 즉흥적(S) | 계획적(P) |
| `R` | 휴양 ↔ 관광 (Rest vs Roam) | 휴양형(R) | 관광형(T) |
| `E` | 외향 ↔ 내향 (Extraversion) | 외향형(E) | 내향형(I) |

> 3개 축 × 2극 = 최대 8가지 조합의 성향 유형 코드(예: `S-T-E`, `P-R-I` …)가 나온다.
> README에 명시된 3축만 우선 확정하며, 축 추가는 별도 이슈로 논의한다(스코프 확장 방지).

---

## 3. 설문 문항 (Questionnaire)

- 축당 **4문항**, 총 **12문항**, **5점 Likert 척도**(1: 전혀 아니다 ~ 5: 매우 그렇다).
- 각 문항은 한쪽 극에 대한 진술이며, 축의 절반은 **역채점(reverse-scored)** 문항으로 응답 편향을 줄인다.

### 3.1 즉흥성 ↔ 계획성 (S)

| # | 문항 | 방향 |
|---|------|------|
| S1 | 여행 전에 일정을 시간 단위로 미리 짜 놓는 편이다 | 정방향(계획) |
| S2 | 숙소나 교통은 출발 직전에 정해도 괜찮다 | 역채점(즉흥) |
| S3 | 예상 밖의 일정 변경이 생기면 스트레스를 받는다 | 정방향(계획) |
| S4 | 여행 중 마음에 드는 곳이 있으면 계획 없이 더 머문다 | 역채점(즉흥) |

### 3.2 휴양 ↔ 관광 (R)

| # | 문항 | 방향 |
|---|------|------|
| R1 | 여행지에서 최대한 많은 장소를 둘러보고 싶다 | 정방향(관광) |
| R2 | 숙소나 자연 속에서 느긋하게 쉬는 것이 여행의 목적이다 | 역채점(휴양) |
| R3 | 하루에 방문할 스팟 개수가 많을수록 만족도가 높다 | 정방향(관광) |
| R4 | 일정이 비어 있어도 굳이 채우려 하지 않는다 | 역채점(휴양) |

### 3.3 외향 ↔ 내향 (E)

| # | 문항 | 방향 |
|---|------|------|
| E1 | 여행 중 만난 낯선 사람과 대화를 먼저 시작하는 편이다 | 역채점(외향) |
| E2 | 혼자 조용히 둘러보는 시간이 꼭 필요하다 | 정방향(내향) |
| E3 | 동행이 많을수록 여행이 즐겁다 | 역채점(외향) |
| E4 | 사람이 많은 장소보다 한적한 장소를 선호한다 | 정방향(내향) |

> ⚠️ E축은 0=외향(E), 100=내향(I)이므로, "정방향" 문항은 항상 **100쪽 극(내향)** 을 진술하는 문항(E2·E4)이고,
> 0쪽 극(외향)을 진술하는 문항(E1·E3)은 **역채점**으로 둔다. S축·R축도 동일한 규칙(정방향 = 100쪽 극 진술)을 따른다.

## 4. 점수 계산 모델 (Scoring Model)

### 4.1 원점수 → 축 점수(0~100)

축별 4문항(정방향 2 + 역채점 2)의 응답을 다음과 같이 정규화한다.

```
정방향 문항 점수      = 응답값(1~5)
역채점 문항 점수      = 6 - 응답값(1~5)

축 원점수(4~20)      = 정방향1 + 정방향2 + 역채점1(변환) + 역채점2(변환)
축 점수(0~100)       = round((축 원점수 - 4) / 16 * 100)
```

- 점수가 **50 초과**면 표에서 "100에 가까움" 쪽 극(P/T/I), **50 이하**면 "0에 가까움" 쪽 극(S/R/E)으로 분류한다.

### 4.2 성향 유형 코드

세 축의 극을 순서대로 이어 `personality_type`을 만든다.

```
personality_type = <S축 극><R축 극><E축 극>
예) 즉흥+관광형+외향  → "STE"
    계획+휴양형+내향  → "PRI"
```

### 4.3 저장 포맷 (`users.personality_scores` JSONB)

```json
{
  "version": 1,
  "axes": {
    "spontaneity": { "score": 72, "pole": "P" },
    "restVsRoam":  { "score": 35, "pole": "R" },
    "extraversion":{ "score": 58, "pole": "I" }
  },
  "answeredAt": "2026-07-06T12:00:00Z"
}
```

`users.personality_type`에는 위 세 `pole`을 이어붙인 `"PRI"` 같은 3글자 코드를 저장한다.

## 5. 매칭에서의 활용 (3-8 선행 참고)

축 점수를 벡터 `(S, R, E)`로 보고, 두 사용자 간 **유클리드 거리**가 가까울수록 성향이 비슷하다고 판단한다.

```
distance = sqrt((S1-S2)^2 + (R1-R2)^2 + (E1-E2)^2)
similarity = 1 - (distance / max_distance)   // 0~1로 정규화, max_distance = sqrt(3 * 100^2)
```

`matches.score`(ERD.md 정의)에 이 `similarity` 값을 저장해 정렬·추천에 사용한다. 실제 알고리즘 구현(지역·거리 필터 포함)은 3-8에서 진행한다.
